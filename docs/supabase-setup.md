# 开通 Supabase 服务指南

## 问题描述

当前系统的数据库同步功能未生效，因为 Supabase 服务还没有开通。

错误信息：
```
数据库初始化失败，使用本地存储: Error: Supabase 环境变量未配置。
请在 Coze 平台开通 Supabase 服务，或设置 NEXT_PUBLIC_SUPABASE_URL 和 NEXT_PUBLIC_SUPABASE_ANON_KEY 环境变量。
```

## 开通步骤

### 1. 登录 Coze 平台

访问你的 Coze 工作台。

### 2. 进入项目设置

1. 找到你的项目
2. 进入项目设置页面
3. 找到"集成"或"服务配置"选项

### 3. 开通 Supabase 服务

1. 在集成列表中找到 **Supabase**
2. 点击"开通"或"启用"按钮
3. 等待服务开通完成（通常需要几分钟）

### 4. 验证开通状态

开通成功后，Coze 平台会自动注入以下环境变量：
- `COZE_SUPABASE_URL` - Supabase 项目 URL
- `COZE_SUPABASE_ANON_KEY` - Supabase 匿名访问密钥
- `COZE_SUPABASE_SERVICE_ROLE_KEY`（可选）- 服务端专用密钥

## 开通后的系统行为

### 自动数据库同步

开通 Supabase 服务后，系统会自动启用数据库同步功能：

1. **工作室信息同步**：
   - 初始化时：从数据库加载工作室信息
   - 修改时：同时更新数据库和 localStorage
   - 降级：数据库不可用时仅使用 localStorage

2. **用户认证同步**：
   - 登录：优先使用数据库验证
   - 修改密码：同步到数据库和本地存储
   - 修改个人信息：同步到数据库和本地存储

3. **数据持久化**：
   - 项目、客户、设计师、案例等数据都会持久化到数据库
   - 支持多端数据同步
   - 支持离线工作（降级到 localStorage）

## 手动验证

开通服务后，重启项目并检查日志：

```bash
# 清理缓存
rm -rf .next

# 重启服务
coze dev
```

查看日志中是否出现：
- ✅ "数据库初始化成功"
- ✅ "工作室信息已从数据库加载"
- ✅ "用户数据已从数据库加载"

## 常见问题

### Q: 开通后还是显示环境变量未配置？

**A**: 重启项目即可。环境变量在项目启动时注入，需要重启才能生效。

### Q: 如何查看当前是否已开通 Supabase？

**A**: 在项目运行时，打开浏览器控制台，查看是否有以下日志：
- `数据库初始化成功` → 已开通
- `数据库初始化失败，使用本地存储` → 未开通

### Q: 可以跳过 Supabase 直接使用 localStorage 吗？

**A**: 可以。系统已经实现了自动降级机制，未开通 Supabase 时会自动使用 localStorage。但如果需要多端同步或数据备份，建议开通 Supabase。

## 数据迁移

如果在开通 Supabase 之前已经使用 localStorage 存储了数据，系统会在首次连接数据库时自动将本地数据同步到数据库。

无需手动迁移，系统会自动处理。

## 技术支持

如果开通过程中遇到问题，请：
1. 检查 Coze 平台的 Supabase 集成状态
2. 查看项目日志：`tail -f /app/work/logs/bypass/app.log`
3. 确认环境变量是否正确注入：`env | grep SUPABASE`
